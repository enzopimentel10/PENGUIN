# Feedback Loops

Feedback loops are the mechanisms through which the PEN.GUIN ecosystem translates the insights generated by the `Learning Engine` into actionable refinements for individual AI agents and their provisioned skills.

## Refining Agent Performance

Agents are not static LLM calls; their operational behavior is shaped by continuous feedback.

1. **In-Session Correction (Short-Term Loop):** This is the immediate feedback provided during the `Execution Loop`. When an agent uses a tool incorrectly, the error message acts as instant negative feedback, prompting the agent to adjust its syntax or approach in the very next turn.
2. **Post-Task Debrief (Medium-Term Loop):** After a task is completed and reviewed, the `Review Agent` or `Test Agent` provides a structured critique. The `Learning Engine` extracts the core lessons from this critique (e.g., "Always null-check deeply nested API responses in React components") and stores it in the agent's specific `Memory Model` profile.
3. **Persona Updating (Long-Term Loop):** When an agent is initialized for future tasks, the `Context Engine` injects the distilled lessons from the Medium-Term Loop directly into the agent's system prompt. Over time, the agent's baseline behavior becomes more robust and aligned with project-specific quirks.

## Refining Skill Performance

Skills, particularly those loaded from `antigravity-awesome-skills`, are also subject to optimization via feedback.

1. **Parameter Optimization:** The `Learning Engine` tracks which arguments agents pass to skills. If it observes that agents consistently have to run a skill multiple times to get the desired output because a default parameter is sub-optimal, it can suggest (to a human operator) or automatically implement a patch to the skill's manifest to adjust default behaviors.
2. **Failure Analysis:** If a specific skill consistently fails or returns unparseable output regardless of which agent uses it, the feedback loop flags the skill in the `Skill Registry` as "Unstable." The `Skill Router` will subsequently deprioritize this skill, preventing it from breaking automated pipelines until it is updated.
3. **Skill Evolution:** By analyzing the "scratchpad" reasoning logs where agents complain about a skill's limitations (e.g., "The `component-generator` didn't include the necessary CSS imports"), the Learning Engine can generate synthetic bug reports. These reports guide the human developers (or specialized Meta-Agents) in expanding the capabilities of the `antigravity-awesome-skills` repository.
