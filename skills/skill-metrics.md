# Skill Metrics

To maintain a high-functioning ecosystem, the PEN.GUIN AI Kernel continuously monitors the performance of all loaded capabilities, particularly those dynamically mounted from `antigravity-awesome-skills`. The system tracks several key metrics to evaluate a skill's empirical reliability.

## Core Performance Metrics

Every time an agent invokes a skill via the `Skill Loader`, the execution is logged and evaluated against the following criteria:

### 1. Success Rate
- **Definition:** The percentage of times the skill successfully completes its intended function without requiring an immediate retry or causing a critical failure.
- **Measurement:** Calculated by dividing the number of successful invocations (exit code 0, parseable output) by the total number of attempts.
- **Importance:** A declining success rate often indicates that a skill is incompatible with a recent change in the ecosystem (e.g., an updated dependency) or that it struggles with certain edge-case inputs.

### 2. Execution Time (Latency)
- **Definition:** The duration it takes for the skill to return a final output after being invoked by the `Skill Loader`.
- **Measurement:** Tracked in milliseconds per execution. The system calculates an average and monitors for significant deviations (P95/P99 latency spikes).
- **Importance:** Slow skills bottleneck the `Execution Loop`. If a skill like `dependency-auditor` suddenly takes 10x longer to run, it flags a potential network timeout issue or an unoptimized algorithmic loop.

### 3. Error Rate
- **Definition:** The frequency of unhandled exceptions, crashes, or unparseable outputs generated by the skill.
- **Measurement:** The ratio of executions resulting in non-zero exit codes or stderr output that the agent cannot automatically recover from.
- **Importance:** High error rates severely degrade agent autonomy, forcing the system into expensive "Error Recovery" loops.

### 4. Review Score (Quality)
- **Definition:** A qualitative metric assessing the actual output of the skill, measured during the `Review & Validation` phase.
- **Measurement:** If the `component-generator` skill successfully outputs React code (High Success Rate, Low Error Rate), but the `Review Agent` consistently flags that code for failing linting standards or accessibility checks, the skill's Review Score drops.
- **Importance:** This ensures that skills are not just "running successfully," but are actually producing high-quality, idiomatic results aligned with `AI_ENGINEERING_RULES.md`.
